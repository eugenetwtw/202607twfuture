# 台灣大未來 — 議題網站 Design Doc

**書名：** 台灣大未來 — 今日瑞士，明日台灣  
**副標／說明：** 與我的競選八十天記錄  
**作者：** 張渝江（Eugene Chang）  
**版本依據：** `2019 台灣大未來-今日瑞士，明日台灣.docx`（2020 紙本／PDF／eBook）  
**狀態：** v1 靜態站已建置於 `site/`（2026-07-12）；本文件仍為產品準則  

---

## 1. 目標

把紙本書轉成**可分享的議題網站**：

- 以**議題（issue）**為最小分享單元，方便貼到各社群平台。
- 每則議題為 **2–3 分鐘**可讀完的精華摘錄，不是整章全文。
- 調性為**個人文集**，非競選官網、非政策白皮書。
- 摘錄引導讀者**借閱或購買完整電子書／紙本**。
- 架構支援**多語系**；URL slug 使用英文，跨語言共用。

### 非目標

- 不做整章全文頁作為分享入口。
- 不做政黨／參選 call-to-action（作者已不再從政參政）。
- 不做留言系統、會員、後台 CMS（第一版）。
- 不做重度 SPA（避免社群爬蟲抓不到 OG）。

## 2. 產品決策（已定案）

| # | 問題 | 決策 |
|---|------|------|
| 1 | 分享粒度 | **議題為主**；不分享整章 |
| 2 | 議題數量 | **較細，約 30+**（可增減，不硬上限） |
| 3 | URL 語言 | **英文 slug**；站台多語系 |
| 4 | 議題頁深度 | **2–3 分鐘精華**（約 400–800 字 + 1–3 圖） |
| 5 | 視覺調性 | **個人文集** |
| 6 | 授權與範圍 | **部份摘錄** + 引導借書／買書 |

## 3. 資訊架構與 URL

### 3.1 路徑規則

```
/{lang}/                         首頁：定位句 + 議題地圖（可篩選主題叢集）
/{lang}/issues/{slug}            議題精華頁（主分享單元）
/{lang}/about                    作者與寫作動機
/{lang}/books                    借書／買書（圖書館、商店、ISBN）
/{lang}/gallery                  （可選）競選／人生紀錄照片選輯
```

- `{lang}`：`zh`（繁中）、`en`（英文，後續）等。
- `{slug}`：英文 kebab-case，**各語言共用同一 slug**。
- 預設語言：第一版建議 `/` 導向 `/zh/`，或 `/` 直接為繁中並提供 `/en/`；實作時二選一寫死於設定。

### 3.2 明確不做的路徑

- `/chapters/...` 全文頁  
- 僅依賴 hash 的 `/#issue-id` 作為主分享連結  

章節僅作議題的**來源 metadata**（例如「來自第 2 章」），不獨立成分享頁。

### 3.3 議題頁區塊（固定版型）

1. **Title** + 一句主張（同時作 OG title／description 基底）
2. **精華正文** 400–800 字 + 1–3 張關鍵圖與圖說
3. **相關議題** chips（3–6 則）
4. **讀全書 CTA**（借書／買書；連結見 §6）
5. **來源註記**：原書章次、書名、作者（非法律全文授權聲明的替代，僅導讀）
6. **Share**：可複製連結；每頁獨立 Open Graph

### 3.4 首頁

- 書名、副標、一句定位（文集，非競選）
- **議題地圖**：卡片牆，可依主題叢集篩選
- 簡短關於作者 + 連到 `/about`、`/books`
- （可選）精選 3 則議題 hero

## 4. 內容模型

### 4.1 Issue（核心實體）

```yaml
id: euthanasia-and-autonomy          # = slug
slug: euthanasia-and-autonomy
cluster: life-health                 # 主題叢集 id
source_chapters: [1, 4]              # 原書章次，僅 metadata
cover_image: media/xxx.jpg           # OG + 頁首圖
related: [healthcare-export, high-value-economy]

# 多語欄位
title:
  zh: 安樂死與生命自主
  en: Euthanasia and Bodily Autonomy
claim:                                 # 一句主張
  zh: "…"
  en: "…"
summary:                               # 可選，列表卡用
  zh: "…"
  en: "…"
body:                                  # 2–3 分鐘正文（Markdown 或結構化 blocks）
  zh: "…"
  en: "…"                              # 缺譯時 fallback zh + 標示「譯文準備中」
figures:                               # 0–3 張
  - src: media/xxx.jpg
    caption:
      zh: "…"
      en: "…"
```

| 原書 | 網站 |

## 8. 視覺與文案調性

### 8.1 要

- 字級偏大、行距鬆、手機優先閱讀
- 圖片當隨筆插圖，帶現場感（工地、市場、旅行、日常）
- 色調偏紙本／墨色／少量風景色
- 第一人稱、觀察、現場細節（符合原書聲音）
- 首頁像**思想地圖／文集目錄**

### 8.2 不要

- 競選藍綠紅、號召入黨、Donate 政治獻金
- 政策白皮書式 dense 表格堆疊為主視覺
- 聳動政論縮圖（可尖銳，但不網軍口吻）

### 8.3 社群分享（OG）

每則 issue 必須有：

- `og:title`：議題中文標題（或當前語言）
- `og:description`：一句主張（可截斷至約 80–120 字）
- `og:image`：該則 cover（建議 ≥ 1200×630 裁切版）
- 絕對 URL（部署網域確定後寫入）

## 9. 技術方針

### 9.1 建議

- **靜態站**（純 HTML/CSS/JS 或極輕量 SSG）
- 議題資料：`content/issues/*.md` 或單一 `issues.json` + 圖片於 `public/media/`
- 由 docx 抽出的 179 張圖：壓縮為 web 用（WebP/JPEG、合適寬度），原圖可另存
- 部署：任意靜態 hosting（GitHub Pages、Cloudflare Pages、Netlify 等）

### 9.2 效能

- 單則議題頁避免載入整本圖庫
- Gallery 分頁或 lazy-load
- 原 docx ~135MB 主因是圖；**站上總體積必須大幅縮小**

### 9.3 版權注意

- 書內聲明：Images copyright by Eugene Chang unless otherwise stated  
- 站上摘錄範圍需符合作者意願；不提供全書可複製全文下載
- 第三方商標、新聞圖等沿用書中既有使用前提；有疑義則換成作者自有照片

1. 任一議題 URL 貼到社群，預覽卡顯示正確標題、主張摘要、代表圖。  
2. 手機上 2–3 分鐘可讀完一則，並清楚知道如何借／買全書。  
3. 首頁可依叢集找到議題，不必懂原書章次。  
4. 站上看起來是**作者文集**，不是競選遺址。  
5. 多語路徑與英文 slug 已就位，後續加 `en` 不必改 URL 結構。
